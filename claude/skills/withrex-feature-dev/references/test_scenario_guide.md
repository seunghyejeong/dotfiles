# 테스트 시나리오 가이드

## 시나리오 작성 형식

```markdown
### TC-[카테고리]-[번호]: [시나리오명]
- **전제조건**: 
- **모드**: 스캔모드 / 취약점모드 / 공통
- **입력**: 
- **실행단계**:
  1. ...
  2. ...
- **기대결과**: 
- **자동화 가능 여부**: Yes / No (사유)
- **우선순위**: P0 / P1 / P2 / P3
- **추가된 기능**: [이 시나리오가 검증하는 신기능명]
```

## 카테고리 코드

| 대분류 | 코드 | 영역 |
|--------|------|------|
| Backend | API | REST API |
| Backend | WS | WebSocket |
| Backend | DB | 데이터베이스 |
| Backend | AUTH | 인증/권한 |
| Backend | SCAN | 스캔 엔진 |
| Backend | VULN | 취약점 엔진 |
| Frontend | UI | 화면 표시 |
| Frontend | UX | 사용자 흐름 |
| Frontend | RT | 실시간 업데이트 |
| Frontend | ERR | 에러 처리 |
| Frontend | VIS | 시각적 검증 (수동) |
| Integration | E2E | 엔드투엔드 |
| Integration | MODE | 모드 전환 |
| Integration | INPUT | 입력 처리 |
| Integration | PERF | 성능 |

## 입력 유형 매트릭스

Phase 0에서 "입력 유형에 따라 동작이 다르다"고 확인된 경우, 아래 매트릭스를 참고해 시나리오를 도출한다.

|           | 스캔모드 | 취약점모드 |
|-----------|---------|-----------|
| 단일 IP   | TC      | TC        |
| 다중 IP   | TC      | TC        |
| 단일 대역  | TC      | TC        |
| 다중 대역  | TC      | TC        |

단, 동작이 동일한 조합은 대표 케이스 1~2개로 충분하다.
"입력 유형과 무관"으로 확인된 경우 대표 입력 1개로만 테스트한다.

---

## 우선순위 기준

| 우선순위 | 기준 |
|---------|------|
| P0 | 핵심 기능 정상 동작, 데이터 손실/보안 문제 |
| P1 | 주요 사용자 시나리오 |
| P2 | 에러 핸들링, 경계값 |
| P3 | 시각적 검증, 성능, 엣지 케이스 |

---

## 작성 예시

```markdown
### TC-API-042: 스캔 결과 필터링 API 정상 응답
- **전제조건**: 스캔 완료 데이터가 DB에 존재함
- **모드**: 스캔모드
- **입력**: GET /api/results?filter=critical (단일 IP 스캔 결과)
- **실행단계**:
  1. 스캔 완료 후 /api/results?filter=critical 호출
  2. 응답 JSON 확인
- **기대결과**: severity가 critical인 항목만 포함된 배열 반환, HTTP 200
- **자동화 가능 여부**: Yes
- **우선순위**: P0
- **추가된 기능**: 결과 필터링
```
