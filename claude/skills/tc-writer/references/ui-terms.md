# UI 용어 표준 정의

## 에러/알림 형태

| 용어 | 정의 | 특징 |
|------|------|------|
| **인라인 에러** | 입력 필드 바로 하단에 표시 | 수동 닫기 불필요, 재입력 시 사라짐 |
| **토스트 메시지** | 화면 특정 위치에 팝업 후 자동 사라짐 | 주로 우상단/하단, 2~5초 후 자동 소멸 |
| **모달 팝업** | 화면 중앙 레이어 팝업 | 뒤 화면 블러 처리, 수동 닫기 필요 |
| **배너 알림** | 화면 상단 전체 너비 배너 | 수동 닫기 필요 |
| **화면 전환** | 다른 페이지/뷰로 이동 | URL 변경 여부 명시 |
| **상태 변경** | 버튼/아이콘/텍스트 상태 변화 | 비활성화·색상 변경 등 |

## 위치 표준 표현

| 표현 | 의미 |
|------|------|
| 필드 하단 | 해당 입력 필드 바로 아래 |
| 화면 우상단 | 브라우저 기준 오른쪽 상단 모서리 |
| 화면 중앙 | 화면 정중앙 (모달용) |
| 헤더 하단 | 상단 네비게이션 바 바로 아래 |
| 콘텐츠 영역 내 | 현재 보고 있는 주 컨텐츠 영역 안 |

## 진입경로 표기 규칙

```
[사이드바/헤더 위치] > [메뉴명] > [탭명] > [버튼명 또는 액션]
```

예시:
- `좌측 사이드바 > [자산 관리] 메뉴 > [취약점] 탭 > [상세 보기] 버튼`
- `상단 헤더 > [설정] 아이콘 > [계정 설정] 메뉴`
- `목록 행 우측 > [...] 더보기 버튼 > [삭제] 옵션`

## 자주 쓰는 전제조건 패턴

```
# 데이터 필요
- 스캔 이력 N건 이상 존재 (N의 근거: [이유])
- 자산이 1개 이상 등록된 상태

# 설정 필요  
- [설정 화면 경로]에서 [설정값]이 [값]으로 지정된 상태

# 권한 필요
- [역할명] 계정으로 로그인된 상태

# 상태 필요
- 스캔 진행 중인 상태 (진행률 > 0%)
- 초기 상태 (데이터 없음)
```
